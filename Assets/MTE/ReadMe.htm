Installation

You can just start using MTE without any installation. But there are something you may want to know.

If you are using Unity3D 32bit, you may need to change the importer setting of MTE.dll. See FAQ: I didn't see the Window/MTE in the menu.

To get a better project organization, please move all folders and files in the Asset\MTE\ to Asset\, (some error will occur but there is no harm) then restart Unity3D. This is required to generate the plugin project separately and to exclude the files in MTE\Editor Default Resources\ from your project's resources.

Important Notice

  • MTE will modify your mesh and related files. Your files may be corrupted under rare conditions. Please, be sure to use a version control system like SVN or GIT to manage your project. It's really important to to back up your files!
  • Do not check Optimize Mesh in the mesh import settings of the obj mesh file when editing with MTE. Such optimization by the mesh importer will possibly change the vertex count and index order. That makes the mesh data inconsistant with the obj file, which leads to many problems when editing the mesh with MTE.
  • Editing operations are undoable before saving the mesh.
  • Please be informed that the unit of brush sizes can be adjusted in Settings/Brush unit size: (actual brush size) = (bursh size) * (brush unit). So, you may need to adjust the brush unit first to a suitable value before starting editing. It's recommended to use the same size as one cell(tile) of the mesh. (The cell size calculated by MTE may be incorrect for meshes not created by MTE.)

Activate MTE Pro

The MTE Pro is licensed per seat. (1 license for each installation of Unity that runs MTE.) You need to activate the product with the invoice number on your machine to use pro-only features.

Activation

  1. Click the about button
  2. Input your invoice number.
  3. Click Activate MTE Pro button, wait a moment. It may take 2-3 minutes, if it still not responds after 3 minutes, please close Unity3D and try again.
  4. If no error occurs, the version description on the top will become: "Version X.X Pro"
  5. You can use all the features of MTE Pro now.

Note One invoice number can activate multiple copies of MTE as long as you bought multiple copies of MTE in the purchase.

Note After reinstalling or upgrading Unity3D, you need to reactivate MTE.

Deactivation

You need to deactivate the product on your machine so it can be activated on another machine.

  1. Click the about button
  2. Click Deactivate MTE Pro button, wait a moment. It may take 2-3 minutes, if it still not responds after 3 minutes, please close Unity3D and try again.
  3. If no error occurs, the version description on the top will become: "Version X.X Free"
  4. You can activate the product on another machine now.

Warning Remember to deactivate MTE Pro before you reinstalling/upgrading your operation system or main hardwares. Main hardware includes the CPU, baseboard and BIOS. Common hardwares like the monitor or keyboard is irrelevant. You will possibly not be able to deactivate the product after the reinstalling and upgrading.

Note You can always deactivate MTE Pro after reinstalling or upgrading Unity3D. Just reactivate MTE Pro on the same machine again and then deactivate.

Note If you can't deactivate MTE Pro online, please contact zwcloud@hotmail.com explaining your situation. Please include the details below:

  • Why can't you deactivate the product online? If your machine was dead, when did that happen? If you manually activated the product, what's the activation code you got?
  • The machine name and user name of the machine. You can find this at Control Panel\System and Security\System. For Mac users, the machine name is at Apple menu/Share.

Then we will manually deactivate the product for you if the situation is acceptable for a manual deactivation. For the same invoice number, you can only deactivate the product in this way once.

User Manual

Mesh Terrain Editor(MTE) has three main functions:

  • The MTE Creator that can create MTE-editable mesh-based terrain.
  • The MTE Converter that can convert Unity3D terrain component into a MTE-editable mesh-terrain.
  • The MTE Editor that can edit a MTE-editable mesh-terrain, including three height tools, a texture painter, a vertex color painter, and a mesh toolbox.

MTE-editable mesh-based terrain will be called mesh-terrain for short. A mesh-terrain is just a mesh with a special terrain shader. MTE provides some shaders with 2/3/4/5 splat textures and two simple vertex-colored shaders. One mesh-terrain only uses one drawcall.

Create mesh-terrains

You can create mesh-terrain in three ways:

  • Create mesh-terrains directly
  • Create mesh-terrains by converting Unity3D terrain
  • Create mesh-terrains by converting an existing mesh

Create mesh-terrains directly

You can create mesh-terrains directly by using the MTE Creator. The MTE Creator can be opened at menu Window/MTE/Creator. Or just right click in the hierchary window and select GameObject -> 3D Object -> Mesh Terrain in the context menu.

Usage steps

  1. Specify the settings
  2. Click Create. It may takes more than 1 minute to finsh if more than 3*3 mesh-terrains are being created.
  3. When it is done, the mesh-terrain is placed in the output dir. And the prefab of converted mesh-terrain is instantiated in the scene.

Create mesh-terrains by converting an existing mesh

If you got a terrain-like mesh that made with 3DSMax or other modeling software, you can convert it into a mesh-terrain using the MTE converter. The MTE Converter can be opened at menu Window -> MTE -> Converter.

Usage Steps

  1. Import your terrain mesh file into a project. Drag it into the hierarchy.
  2. Open the MTE Converter and click Mesh.
  3. Select the mesh GameObject(the one with a MeshRenderer component)
  4. Specify the settings.
  5. Click Begin Converting and wait a moment. It should finish in about 10-30 seconds.

Use an existing mesh (experimental feature)

Requirements for height tools:

  • The mesh is an obj or asset file.
  • The mesh uses +Y as it's height growing direction, namely y-up.
  • The mesh GameObject in the scene has a valid MeshFilter and MeshCollider attached.

Requirements for texture painter and vertex color painter:

  • The mesh file should contain a layer of uniformly spread uv.
  • The min/max point of the bounding box in x and z should have the min/max uv. min uv: (0,0), max uv: (1,1).
  • The mesh object in scene uses a similiar shader with those provided by MTE. It is highly recommended that use a material created by MTE creator instead. See also FAQ: Can I change the shader?

Requirements for mesh toolbox:

  • The mesh is an obj or asset file.

To convert your mesh to one that can be edited with MTE, just use the tools provided by MTE.

  • Q: What if I'm using FBX or other formats that MTE does't support?
    A: Use Mesh Saver.

    Mesh Saver is a tool that can save a single mesh as *.obj or *.asset file. It is located at menu item Windows -> MTE -> Tools -> Mesh Saver.

    1. Drag your mesh object in scene to MeshSaver's window.
    2. Choose a file format you want to use: *.obj or *.asset.
    3. Click SaveAs to save the mesh in the mesh object to another file.

  • Q: What if my mesh is z-up?
    A: Use Mesh Z-up to Y-up Converter.

    Mesh Z-up to Y-up Converter is a tool that can convert a z-up mesh to a y-up one. It is located at menu item Windows -> MTE -> Tools -> Mesh Z-up to Y-up Converter

    1. Drag your mesh file, or exactly the mesh inside it, in the project window to the converter window.
    2. Click Begin Converting to convert the mesh to y-up.
    3. Click apply to save the mesh to file. Or, if you find something wrong, Click revert to cancel the modification. Again, please always remember to use a version control system to back up your file!

    It just rotates the mesh -90 degrees around z axis. And this operation doesn't work for all kinds of z-up meshes. You can try to rotate the mesh in scene view by assign rotation to (-90, 0, 0) to see if it is y-up after that. If it is, then this converter will work for your mesh.

  • Q: What if my mesh's uv is not uniformly spread from (0,0) to (1,1)?
    A: Use Mesh UV Adjuster.

    Mesh UV Adjuster is a tool that can normalize uv of a mesh and invert u or v of a mesh. (uv means the texcoord of a mesh.) It is located at menu item Windows -> MTE -> Tools -> Mesh UV Adjuster

    1. Drag your mesh file, or exactly the mesh inside it, in the project window to the adjuster window.
    2. Click Normalize to normalize uv: spread uv uniformly from (0,0) to (1,1) on the mesh.
    3. Drag your mesh to the scene. Create a temp mesh-terrain and use its material for your mesh. Then try using the texture painter to paint on your mesh object. If the painting position is different from the mouse position. Try Inverting u or v of it:
      • Click Invert U to invert u of mesh texcoord: u becomes 1 - u.
      • Click Invert V to invert v of mesh texcoord: v becomes 1 - v.
    4. Click apply to save the mesh to file. Or, if you find something wrong, Click revert to cancel the modification. Again, please always remember to use a version control system to back up your file!

  • Q: What if I want to adjust the pivot point of a converted mesh?
    A: Use Mesh Pivot Adjuster.

    Mesh Pivot Adjuster is a tool that can offset the pivot point, the (0,0,0) point in mesh space, of a mesh. It is located at menu item Windows -> MTE -> Tools -> Mesh Pivot Adjuster

    1. Drag your mesh file, or exactly the mesh inside it, in the project window to the adjuster window.
    2. Click one of the offset buttons to move mesh towards specific directions by its aabb box size in that axis.
    3. Click apply to save the mesh to file. Or, if you find something wrong, Click revert to cancel the modification. Again, please always remember to use a version control system to back up your file!

Edit mesh-terrains

The height, textures and vertex colors is editable for a mesh-terrain. MTE also provided a mesh toolbox to move mutiple vertices freely or remove vertices of the mesh.

The Filter

The mesh-terrain filter is used to help editing multiple editing together.

Usage: just set the layer and/or tag of the mesh-terrains, done.

When you add a new mesh into the scene, you need to click the filter button to refresh manually.

Only mesh-terrains that match the filter can be edited. The splat-textures used by those mesh-terrains will be collected into a texture list used by the texture painter.

MTE Free users can only edit first two mesh-terrains together.

Height Tools

Use first three height-tools to modify the height of vertices of the mesh-terrain.

  • Raise and lower


    Use mouse to raise and lower height in the scene view. Press Space to toggle between raising up and lowering down.

    You can modify the size and strength of the brush in the editor window.

    You can also use -/+ to decrease/increase the size of the brush and [/] to decrease/increase the strength.

    You can adjust the falloff curve using the curve editor.

  • Paint height


    Hold Ctrl and move mouse to sample a height. And use mouse to paint height in the scene view.

    The height can also be specified directly in the editor window.

    You can modify the size and speed of the brush in the editor window.

    You can also use -/+ to decrease/increase the size of the brush and [/] to decrease/increase the speed.

  • Smooth height


    Use mouse to smooth height in the scene view.

    You can modify the size and speed of the brush in the editor window.

    You can also use -/+ to decrease/increase the size of the brush and [/] to decrease/increase the speed.

Warning Save the mesh terrain by clicking Save below! Ctrl+S WONT work!

Paint textures

When the filter is refreshed, all splat-textures used by mesh-terrains will be collected into the texture list.

Select a brush and a texture from the texture list, then click to draw on the mesh-terrains in the scene view. Only mesh-terrains using the selected texture can be painted. You can change the splat-textures in the material editor of a mesh-terran.

You can modify the size and flow of the brush in the editor window; also you can use -/+ to increase the size of the brush and [/] to decrease/increase the flow.

You can add or remove the brushes by adding/removing the pictures in Editor Default Resources\MTE\Brushes. The import settings for those brushes should be like this:


Then click to refresh the brushes.

Related tools:

  • Basemap Baker

    Window -> MTE -> Tools -> Basemap Baker

    Bake the material of the selected mesh-terrain into a png picture. A basemap in png format will be generated next to the mat file of the mesh-terrain.

    A base-map is a lower resolution composite image of the blended splat-textures. See this post for more information.

  • Mesh to Terrain Converter

    Window -> MTE -> Tools -> Mesh to Terrain Converter

    Convert a mesh-terrain to Unity3D terrain. A Unity3D terrain asset file will be output to Assets.

Modification to the control texture(s) will be saved instantly.

History, Undo and Redo

All editing operations on the mesh or textures are undoable. Use PageUp to undo and PageDown to redo. Changes to the parameters on Editor GUI are not undoable.

You can use the simple history viewer(at menu Window/MTE/Tools/History Viewer) to inspect your editing histroy.

Note On Mac, use Fn+ to undo, Fn+ to redo

Note Undo/redo data will be cleared after you saving the mesh.

Customize UI Language

The default language is English. But you can adjust the language and text content of the GUI.

  • Change the Language:

    Go to MTEEditor -> Settings -> Language and change it. It will close all MTE windows and recompile your scripts. After that, the language is changed to the one you just selected.

  • Change UI Contents:
    1. Locate string table files in Editor Default Resources\MTE\Languages\. There are two string table files: en-US.xml and zh-Hans.xml. Choose one you want to modify.
    2. Modify string file: For example, modify <Warning_Confirm>Are you sure?</Warning_Confirm> to <Warning_Confirm>You need to confirm this operation! Are you sure?</Warning_Confirm>
  • Use a new language:
    1. Copy en-US.xml and rename it to, for example, ja-JP.xml or es-ES.xml.
    2. Translate the string in the copied string table.
    3. Reimport MTE.dll.
    4. After the import is finished. Go to MTEEditor -> Settings -> Language and change it to your language.

FAQ

  1. I didn't see the Window/MTE in the menu

    Please open the importer setting of Plugins/Editor/MTE/MTE.dll. Then check if the Platform Settings->CPU is AnyCPU or corresponds to the using Unity3D. If you're not sure, just try changing that.

  2. Why pressing Ctrl+Z will not redo?

    Use PageUp to undo and PageDown to redo. These keys are used instead of the famous Ctrl+Z because the undo system provided by Unity3D is just TOO SLOW for big data such as a big array. Ill try to make Ctrl+Z/Y work in future versions, maybe.

    Please click the *keyboard button* on the top of MTE editor to confirm hotkeys.

  3. I cannot edit using the height tool. Not seeing any flashing vertex.

    Try disabling and enabling the `MeshCollider` component. We still cannot figure out why it happens and why this works. This problem may happen when you reimport/save the mesh-terrain obj file.

  4. Can I change the shader?

    Sure. And all editing functions have been decupled from the shaders provided by MTE. But if you want to continue editing the mesh-terrain in MTE, you must follow conventions below.

    • To use the height tools(raise/lower, paint height, smooth height), make sure that the object has a valid mesh and MeshCollider attached.
    • To use the paint textures tool, make sure that the shader contains a Texture2D property named _Control and it is all red in the beginning. If you use a second control texture, the property name of it must be _ControlExtra and it is all black in the beginning. But I suggest you use the shaders provided by MTE as templates. The splat textures' property name should be _Splat0/1/2/3/4. If you are using a bumped shader, the property name of the normal textures should be _Normal0/1/2/3/4. We suggest you use the shaders provided by MTE and modify them as you see fit.
    • To use the paint vertex color tool, make sure that the gameObject selected have a VertexColorInitializer component attached. (This script component is automatically attached to the gameObject using the button on the editor window. See the description of this tool.)
    • To delete vertices with the mesh toolbox, make sure that the gameObject selected does NOT have a VertexColorInitializer component attached: you cannot cut holes on a vertex-colored mesh-terrain.

  5. Hotkey not working!

    Does the scene view or the MTE editor has the focus? If it is not, the hotkey will not work.

    Do you close the IME? Its the IME that is disturbing the hotkey handling routine and makes it not work. Close it.

  6. I cannot unselect the mesh-terrain. / I cannot select other gameObjects.

    Please select other gameObject in the hierarchy window. MTE locked the selection in the scene view window to forbid misoperation when editing the mesh-terrain.

  7. What is done when adding vertex color component?

    This operation will create a asset file of the vertex colors aside the mesh file. And a VertexColorInitializer component will be attached to the gameObject.

    The component is used to apply vertex colors in the asset file to Mesh.colors when the gameObject is created. The source codes of the component and the asset are provided here: Assets\Editor Default Resources\MTE\MTE.zip. The cs files inside MTE.zip have been already compiled into a dll here: Assets\Plugins\MTE\MTE_script.dll. The source codes are provided so you can modify them and compile them as MTE_script.dll and then replace the original one if you want to. Don't extract them into your project directory.

  8. Why MTE Editor is closed when in play mode?

    Editing in play mode is not tested yet. So MTE editor is just closed before entering play mode to restrict possible misoperations.

  9. Why there is a Material folder in the output dir when conversion finished?

    Its generated by Unity for previewing the mesh file in the inspector. You can safely remove it yourself.

  10. The MTE Editor/Converter window disappered when it loses focus on MacOS.

    This is by design of Unity3D and Apple. See also this post.

  11. Can MTE paint more than 5 splat textures?

    Yes. But limited. See this post for details.

    By the way, another plugin called MegaSplat supports painting with 256 textures and the developer are trying to integrate MTE texture painting functions with MegaSplat. This feature should be available in a future version.

Known Issues

  1. CPU occupancy rate is high when using height tools for some big mesh-terrain.

    The hit point of mouse is generated using Physics.Raycast, so the MeshCollider needs to be updated each time the Mesh is modified. It is the process of updating a collider, which is handled by Unity3D internally, that consumes much CPU time when editing heights of a mesh with many vertices.

    This problem may also happen even when the number of vertices is not big. It can be solved by manually re-compiling your scripts or reimporting MTE.dll. Remember to save your changes first!
